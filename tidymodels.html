Why tidymodels? - 1

The tidymodels framework in R is a collection of packages for modeling and machine learning that share the underlying design philosophy, grammar, and data structures of the tidyverse. Here are some of the key reasons to use tidymodels:

  1. Tidyverse Integration: If you’re already familiar with the tidyverse, it can make sense to use tidymodels because the syntax, structure, and style will be consistent. This can make your code more readable and efficient.

  2. Modularity: tidymodels is designed to be modular, which means that different parts of the modeling process (such as pre-processing, modeling, and validation) are handled by separate but compatible packages. This enables a high degree of customization and flexibility.

  3. Preprocessing: The recipes package within tidymodels offers a great deal of flexibility for preprocessing tasks, such as feature engineering, imputation, and data scaling, which are crucial in many machine learning workflows.

  4. Unified Interface: tidymodels offers a unified interface to different types of models, including regression, classification, and unsupervised models, so you don’t have to learn new syntax for each model you want to try.

Why tidymodels? - 2

  1. Workflow Management: tidymodels includes the workflows package which can bundle together your pre-processing, modeling, and post-processing steps. This can make your code more organized and easier to maintain, especially for complex analyses.

  2. Resampling: With the rsample package, tidymodels provides infrastructure for efficient resampling, such as cross-validation and bootstrap sampling, which are essential techniques in machine learning to avoid overfitting and to provide reliable model assessments.

  3. Tuning: The tune package allows you to optimize model hyperparameters, which can substantially improve model performance.

  4. Model Explanation: The vip and pdp packages in tidymodels allow for effective visual interpretation of models, making it easier to explain your models’ predictions.

Preprocessing data

Recipe functions - 1

The recipes package in R is designed to create and preprocess variables for machine learning algorithms.

  1. recipe(): This function is used to initiate a recipe, where you provide the formula and data that you will be using.

  2. prep(): After defining a recipe, this function is used to estimate the parameters from a training set that can be later applied to other data sets.

  3. bake(): After prep(), this function is used to create the final design matrix.

  4. juice(): This function can be used after prep() to get the preprocessed training set.

Recipe functions - 2

  1. step_*(): These are a variety of functions used to specify what preprocessing steps should be added to the recipe. Some examples are:

    • step_center(): This function centers (mean subtracts) numeric variables.

    • step_scale(): This function scales (divides by the standard deviation) numeric variables.

    • step_dummy(): This function creates dummy variables from a factor variable.

    • step_log(), step_sqrt(): These are various transformations you can apply to variables.

  2. add_role(): Add/replace roles to variables in the recipe.

  3. update_role(): Update variable roles and types.

  4. check_recipe(): Check the steps of a recipe for possible issues.
